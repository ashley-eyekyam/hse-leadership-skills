# Cross-list — Rail signalling / track-safety (possession & protection) map

**Fragment ID:** `KB-REG-RAIL-SIGNALLING`
**What this is:** a copyright-safe **task → protection-arrangement → artifact map**
for safe work on or near the rail line — the per-facet content cross-listed onto
the rail sector pack's permit and safe-work skills. It names the on-track task
classes, the line-protection arrangement each requires, and the artifact produced,
read alongside the SMS map (`KB-REG-ROGS`) and the safe-system-of-work practice.
**What this is NOT:** a reproduction of ROGS, the Rule Book, or any standard text.
Cite the regulation/topic only — never paste standard or rule-book wording.

## On-track task → protection arrangement → artifact it grounds

| On-track task | Protection arrangement (paraphrased) | Artifact it grounds |
|---|---|---|
| Work in a possession | Take/grant a possession; block the line | possession arrangement + safe-work pack |
| Work with line open (lookout) | Lookout / warning system + position of safety | safe-system-of-work plan (SSOW) |
| Work near electrified line (OLE/3rd rail) | Isolation / earthing + clearance | electrical isolation + permit |
| Signalling alteration / commissioning | Signal-engineering safe-isolation + testing | signalling testing/commissioning record |
| Level-crossing / public interface | Protect the crossing during work | crossing-protection arrangement |
| Adjacent-line open | Adjacent-line working controls | ALO risk assessment + briefing |

## How the skill uses this fragment
- A rail safe-work skill resolves the on-track task first, grounds the protection
  arrangement here, references `KB-REG-ROGS` for the SMS/competence hook, and
  produces the SSOW / possession artifact.
- Controls follow the hierarchy of controls (separate work from trains before
  relying on lookout); a missing input is `[GAP]`. `bundled_in` wiring deferred to
  Phase 17 (SEAM-01).
