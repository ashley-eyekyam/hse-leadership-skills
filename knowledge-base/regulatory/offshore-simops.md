# Cross-list — Offshore simultaneous-operations (SIMOPS) map

**Fragment ID:** `KB-REG-OFFSHORE-SIMOPS`
**What this is:** a copyright-safe **concurrent-activity → interaction-hazard →
control map** for simultaneous operations (SIMOPS) on offshore installations —
the per-facet content cross-listed onto the marine/offshore sector pack's permit
and safety-case skills. It names the concurrent-activity pairs, the interaction
hazards they create, and the SIMOPS-matrix control each demands, read alongside
the safety-case map (`KB-REG-OFFSHORE-SCR`).
**What this is NOT:** a reproduction of any standard, OGUK/IOGP guidance, or
safety-case text. Cite the activity/control topic only — never paste standard
wording.

> Source: offshore SIMOPS practice (cross-walks KB-REG-OFFSHORE-SCR safety-case regime + sector SIMOPS-matrix practice) — practice map · Year: 2026 · Reviewed: 2026-06-21 · Volatile: true (sector guidance drift; refresh on review).

## Concurrent-activity pair → interaction hazard → control it grounds

| Concurrent activities | Interaction hazard | SIMOPS control it grounds |
|---|---|---|
| Drilling + production | well-control vs. hydrocarbon inventory | SIMOPS matrix cell + isolation/standby |
| Lifting (crane) + personnel transfer | dropped object over personnel | exclusion zone + suspended-ops trigger |
| Diving + dynamic positioning / thrusters | diver near active thrusters | thruster lock-out + dive-control interface |
| Hot work + hydrocarbon handling | ignition vs. release | hot-work permit gating + atmosphere control |
| Helicopter ops + crane ops | airspace conflict | radio-controlled suspension of conflicting op |
| Bridging (multiple duty-holders) | interface gaps | bridging document + combined SIMOPS matrix |

## How the skill uses this fragment
- An offshore permit/safety-case skill enumerates the concurrent activities,
  grounds the interaction hazard and the SIMOPS-matrix control here, and references
  `KB-REG-OFFSHORE-SCR` for the safety-case hook.
- Controls follow the hierarchy of controls (separate in time/space before relying
  on procedure); a missing input is `[GAP]`. `bundled_in` wiring deferred to
  Phase 17 (SEAM-01).
