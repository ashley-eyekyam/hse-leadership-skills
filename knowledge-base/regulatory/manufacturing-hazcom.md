# Cross-list — Manufacturing hazard-communication (HazCom / GHS) map

**Fragment ID:** `KB-REG-MFG-HAZCOM`
**What this is:** a copyright-safe **duty → artifact map** for hazard
communication in a manufacturing setting — the per-facet content cross-listed onto
the manufacturing sector pack's chemical-handling, SDS and labelling skills. It
names the HazCom program duties (inventory, SDS access, labelling, training) and
the artifact each produces, read alongside the jurisdiction map (US 29 CFR
1910.1200 HCS / EU CLP `KB-REG-EU-CLP`) and the GHS standard (`KB-STD-GHS`).
**What this is NOT:** a reproduction of the HCS, CLP, or GHS text. Cite the
CFR/regulation/topic only — never paste standard wording.

> Source: hazard-communication practice (cross-walks OSHA HCS 29 CFR 1910.1200 + EU CLP (EC) 1272/2008 + GHS) — practice map · Year: 2026 · Reviewed: 2026-06-21 · Volatile: false (stable program structure; refresh on review).

## HazCom duty → artifact it grounds

| Duty (paraphrased) | Artifact it grounds |
|---|---|
| Maintain a hazardous-chemical inventory | chemical inventory / register |
| Obtain + make SDS accessible | SDS library + access procedure |
| Label containers (workplace + shipped) | container-labelling scheme (GHS pictograms/signal-word) |
| Written HazCom program | the HazCom program document |
| Worker training on hazards + protective measures | training records |
| Non-routine task / process-chemical hazards | task-specific exposure controls |

## How the skill uses this fragment
- A manufacturing chemical/SDS skill grounds the program duties here, cross-walks
  the GHS classification (`KB-STD-GHS`) and the jurisdiction map for the binding
  form/label, and applies the hierarchy of controls to exposure.
- De-identify any worker exposure data per the mandatory de-id block; a missing
  input is `[GAP]`. `bundled_in` wiring deferred to Phase 17 (SEAM-01).
