# Defensibility Framing — make every conclusion survive a regulator's challenge

**Fragment ID:** `KB-SNIP-DEFENSIBILITY`
**Prompt text.** Encodes the project's durable invariant: no vague controls, no
PPE-only treatments, every conclusion traced to evidence with named owners and dates.

> Source: project core value (CLAUDE.md) + reasonably-practicable / SFAIRP doctrine · Year: 2026 · Reviewed: 2026-05-15 · Volatile: false.

---

## Instruction

Before finalising any artifact, check it would survive a competent challenge:

1. **Specificity** — every hazard, control, and finding is task/site/asset-specific,
   not generic. Reject "be careful", "follow procedure", "wear PPE" as standalone.
2. **Hierarchy of controls** — every control ranked (`KB-SNIP-HOC`); any lower-order-
   only control justified-or-escalated.
3. **Evidence trail** — every conclusion traced to its evidence (observation,
   measurement, citation). Figures carry `source`+`year`. Statutory claims carry the
   jurisdiction reference (resolved via `../regulatory/`), never fabricated text.
4. **Proportionality** — where the jurisdiction uses a "reasonably practicable" /
   SFAIRP test, state the proportionality reasoning explicitly.
5. **Accountability** — every action has a **named owner** and a **due date**; every
   residual risk is stated, not hidden.
6. **Assumptions surfaced** — `[ASSUMPTION]` / `[GAP]` flags from intake remain
   visible; nothing invented to look complete.

An artifact that fails any of these is not yet defensible — fix it before delivery.
