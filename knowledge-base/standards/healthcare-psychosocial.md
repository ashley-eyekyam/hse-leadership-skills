# Healthcare psychosocial risk — cross-list into ISO 45003 (topic→artifact map)

**Fragment ID:** `KB-STD-HEALTHCARE-PSYCHOSOCIAL`
**What this is:** a copyright-safe **healthcare-context cross-list** that points the
sector-specific psychosocial hazards (workplace violence, emotional labour, shift
fatigue, moral distress) at the existing ISO 45003 work-design-control structure.
It does NOT re-author the standard map — it references `KB-STD-ISO45003`.
**What this is NOT:** a reproduction of ISO 45003 text, nor a duplicate of the
shipped psychosocial map. The structural reference lives in `iso-45003.md`; this
fragment is the healthcare lens onto it.

> Source: ISO 45003:2021 (referenced via KB-STD-ISO45003) + healthcare-sector psychosocial-hazard context · Year: 2026 · Reviewed: 2026-05-15 · Volatile: false.

Healthcare work carries distinctive psychosocial hazards. This cross-list maps each
to the ISO 45003 topic that grounds its assessment and control — applying the
**work-design-control, not individual-resilience-only** discipline from
`KB-STD-ISO45003` to the healthcare setting. (Content now; the `bundled_in` seam
wiring into the sector pack is deferred to Phase 17.)

## Healthcare psychosocial hazard → ISO 45003 topic it routes to

| Healthcare psychosocial hazard | ISO 45003 topic (see `KB-STD-ISO45003`) | Work-design control it grounds |
|---|---|---|
| Workplace violence & aggression (patients/visitors) | psychosocial hazard ID — work environment / social factors | de-escalation design, staffing, layout, security (not "be resilient") |
| Emotional labour / compassion fatigue | psychosocial hazard ID — aspects of work organisation | caseload design, supervision, debrief structures |
| Shift work & fatigue | psychosocial risk assessment | roster design, fatigue management, recovery time |
| Moral distress / ethically hard decisions | psychosocial hazard ID — social factors | escalation pathways, ethical support, role clarity |
| Bullying / incivility | psychosocial hazard ID — social factors | civility standards, reporting, leadership accountability |

## Usage note
- Resolve psychosocial structure through `KB-STD-ISO45003` (this fragment does not
  duplicate it); apply `KB-SNIP-HOC` so controls are work-design-first.
- Healthcare physical/biological hazards (sharps, IPC, biosafety) live in the
  hazard-library fragment `KB-HAZ-HEALTHCARE`, not here.
- Pair with `KB-STD-ISO45001` (45003 extends it). Missing context is `[GAP]`, never invented.
