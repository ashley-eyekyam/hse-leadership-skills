# Pre-output Quality Checklist — Construction Phase Plan (CDM 2015 Reg 12)

Validate the draft CPP against this gate before producing any output (markdown or a rendered
DOCX/PDF). Any unchecked item is fixed and the draft re-validated.

## Specificity & the GATE
- [ ] The **project is named and specific** (scope + programme) — **not** a generic "a building
      site"; ≥1 significant activity and the contractor configuration are captured.
- [ ] The plan is **proportionate** to the project's risk (a small refurb's plan is short; a
      multi-contractor new-build's is fuller) — not a boilerplate template.

## Significant risks & controls by activity (the core lever)
- [ ] **Every significant activity** from intake has a row with a **hierarchy-ranked control +
      a named owner**; every control walked Elimination → Substitution → Engineering →
      Administrative → PPE.
- [ ] **Work at height leads with collective protection** (edge protection / MEWP / nets /
      elimination) — a "operatives to wear harnesses" control with no collective protection is
      **flagged PPE-led and pushed up the hierarchy** (or an explicit justification recorded);
      never the headline control.
- [ ] Residual risk is **re-scored after controls**; a residual High/Critical carries a
      hold-point or further controls.

## Notification & duties
- [ ] The **F10 / Reg 12 notification position is stated** for a notifiable project (a
      notifiable project that omits it is a citation defect).
- [ ] The **sole contractor's Reg 12(2)** ownership of the CPP is honoured (the duty is not
      excused).

## CDM chain & assumptions (loose coupling)
- [ ] The **one-line PCI → CPP → H&S File cross-reference** (Reg 4 → 12 → 12(5)) is present,
      sourced from `KB-SNIP-CONSTRUCTION-CLAUSE-MAP`, **without assuming a sibling skill ran**.
- [ ] Where the **PCI was absent**, the Reg 4 gap is flagged `[GAP]` with the **assumptions**
      the plan proceeds on — no invented input.

## Citation, India, actions, de-id
- [ ] Every citation traces to the KB (CDM 2015 Reg 12 / L153 for UK; 29 CFR 1926 Subpart C for
      US); for **India** the **state is resolved** and a literal `[GAP]` is emitted where a
      BOCW state form/return is owed — **never a minted national form number**.
- [ ] Every action carries a **named owner + an ISO due date + a hazard link** (no anonymous
      actions, no "ASAP").
- [ ] **De-identification pass complete BEFORE drafting**; no input-derived identifier, no
      named injured operative, no embedded re-id key, no unsuppressed `<5` injury cell in the
      circulated plan (the user-supplied duty-holders stay named — the legitimate exception).
- [ ] The **SME Review & Sign-off** ran; the output **never claims competent-person approval**
      on this statutory-adjacent CPP.
