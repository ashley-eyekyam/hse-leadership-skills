# Pre-output Quality Checklist — Contractor Prequalification

Validate the draft against this gate **before** producing any output. The first three items
are the core-value integrity gate — a failure there is not a style nit.

## Evidence-not-claims (the integrity gate)
- [ ] **Every score is tied to a verified evidence item** — no criterion is scored "met / pass"
      on a self-asserted claim. An accreditation or competence claimed but **not evidenced** is
      recorded as `[GAP]`, never as "met".
- [ ] **Every `[GAP]` blocks an unconditional pass** — no `approve` recommendation is issued
      over an open `[GAP]`; it is `conditional` (with a closing action) or `reject`.
- [ ] **High-hazard (T3) work is not passed on PPE/competence claims alone** — higher-order
      controls / activity-specific certification are evidenced (`KB-SNIP-HOC` applied).

## Risk-tier proportionality
- [ ] The **risk tier is set by the highest-risk activity** in the scope, not the average — a
      single hot-work / confined-space / height / lifting / energised-electrical / demolition
      task lifts the package to T3.
- [ ] The **PQQ depth and pass threshold match the tier** (`KB-DATA-CONTRACTOR-TIERS`): high-hazard
      work demands the deeper PQQ + higher bar than low-risk admin work.

## Defensibility
- [ ] The recommendation is **approve / conditional / reject** with a clear rationale traced to
      the verified evidence register.
- [ ] **Conditions are real SMART actions** — named owner + ISO due date + measure, each tied to
      the gap it closes; **a review date** is set (from evidence expiry or the client cadence).
- [ ] The right statutory regime is cited accurately — **ISO 45001 8.1.4** always; **CDM 2015
      reg. 8** where construction; the **OSHA multi-employer policy** for a US controlling-employer
      scenario; for India, the resolved state record (never a national form number).

## De-identification (ran FIRST)
- [ ] The de-identification pass completed **before scoring**; **no named injured person** from
      the contractor's accident/enforcement history reaches the scorecard.
- [ ] **No injury cell < 5** is published un-suppressed; no re-identification key is embedded in
      the output.

## Specificity
- [ ] The pack names the **actual contractor + work package + site** — not "a contractor / general
      safety". A generic, copy-paste-flavoured PQQ that could apply to any contractor is a defect.
