# Pre-output Quality Checklist — Safety Culture Assessment

Before producing any output, validate the draft against this gate:

- [ ] **De-identification pass complete BEFORE drafting** — identifiers listed up front;
  no residual identifier, no re-id key embedded; no individual named as "the culture
  problem".
- [ ] **No <5-cohort cell published** — every cohort/sub-group breakdown has ≥5
  respondents; secondary suppression applied; no identifying verbatim quote reproduced.
- [ ] **No rating from a single survey** — any maturity rating draws on a named model, a
  **named population**, and **≥2 triangulated data sources**; a single-survey rating is
  refused (record `[GAP]`).
- [ ] **Schein read as a gap diagnosis, not a band** — standalone Schein output names the
  espoused-vs-enacted gaps; no "Schein Level N" maturity rating.
- [ ] **Model fidelity** — each banded result quotes its model `source`+`year`
  (`KB-DATA-CULTURE-MATURITY`); the culture models are cited as **method, not law**; no
  fabricated band.
- [ ] **Systemic advancement above awareness** — advancement controls are systemic
  leadership-system / work-design changes (via `KB-SNIP-HOC`); no poster-campaign-only or
  "tell people to care more" roadmap.
- [ ] **Advancement tracked to leading indicators** — every roadmap action has a named
  role-label owner + an ISO due date + a tied leading indicator from
  `KB-DATA-LEADING-INDICATORS`.
- [ ] **Sibling cross-reference present** — the output points to `bbs-program-designer`,
  `safety-walk-gemba`, and `leading-lagging-kpi-framework` to operationalise the roadmap.
- [ ] **Branded report assembled** — `report.json` built and rendered with a
  confidentiality statement section.
