# HSE Leadership Skills - Skill Manual

Welcome to the **complete reference for the v1.2 HSE Leadership Skills catalog**. The project has 94 total catalog entries. This Wiki documents the 93 non-forge manual entries - the 92 consultant artifact skills plus the `using-hse-skills` guided router - and links `hse-skill-forge` separately as a contributor tool.

> **New here? Start with the router.** Run **/using-hse-skills** when you are new, unsure which skill to use, or facing a multi-step HSE task. It recommends one skill or an ordered chain, then hands over the context it captured so the next skill starts warm. See **[Start Here - Guided Mode](Start-Here-Guided-Mode)**.

This is the **reference catalog**. It does not restate one-time setup or the deep flagship walkthroughs — for those, see [Getting Started](../docs/USER_MANUAL.md). For chaining several skills into a real end-to-end task, see [End-to-end journeys](../docs/USER_JOURNEYS.md).

## How to read a skill card

Every full card on a pack page uses the same shape. The fields mean:

- **Produces** — the artifact the skill creates, in one line.
- **For** — the audience the skill is written for, using the legend below.
- **Grounded in** — the standard or jurisdiction the skill cites (e.g. ISO 45001 cl. 6.1.2).
- **Packs** — the install bundle(s) the skill ships in (a shared skill lists every pack it belongs to).
- **Use when** — the trigger situation the skill is built for.
- **Don't use for** — the boundary, redirecting you to the correct skill instead.
- **Have ready** — the inputs the skill's intake interview will demand before it proceeds.
- **Trigger** — representative phrases that automatically wake the skill.
- **You get** — the report sections / output structure you receive.
- **Pairs well with** — a complementary skill or journey to reach for next.

**Audience legend:** **M** = manager · **E** = executive · **F** = frontline · **C** = consultant.

## The five universal rules

These five behaviours are baked into **every** skill in the pack. The full treatment is in [Getting Started — Part 2](../docs/USER_MANUAL.md#part-2--five-rules-that-apply-to-every-skill); the one-line recap:

1. **Be specific, or it won't proceed** — every skill interviews you and refuses vague answers; name the exact task, steps, and site. [Details »](../docs/USER_MANUAL.md#rule-1--be-specific-or-it-wont-proceed)
2. **Hierarchy of controls — PPE is the last line** — controls are ranked Eliminate → Substitute → Engineer → Admin → PPE; a PPE-only fix with no justification is a defect. [Details »](../docs/USER_MANUAL.md#rule-2--the-hierarchy-of-controls-is-enforced)
3. **De-identification runs first, key kept separate, leak = auto-fail** — personal/health data is scrubbed to role labels before drafting, the re-identification key is handed back separately, and any leak is a non-waivable failure. [Details »](../docs/USER_MANUAL.md#rule-3--personal-data-is-de-identified-automatically-and-this-is-a-hard-rule)
4. **Decision-support only — competent-person sign-off required** — every output is a draft for review; regulatory references are pointers to verify against the current authoritative source. [Details »](../docs/USER_MANUAL.md#rule-4--decision-support-only-a-competent-person-must-sign-off)
5. **Output = branded PDF + Word** — findings render to both a branded PDF and an editable Word document; the Eyekyam theme is the default and you can swap in your own. [Details »](../docs/USER_MANUAL.md#rule-5--output-comes-as-a-branded-pdf--word-document)

## The packs

The 94 skills are organised into **16 consultant bundles**. The catalog total is 94 skills; the `hse-all` "install everything" bundle carries 93 of them (it includes the `using-hse-skills` router — which also ships in `hse-core` — so newcomers get the navigator, and omits only the `hse-skill-forge` authoring tool, which ships in `hse-systems`). Pick a pack to see its full cards.

**Core and the two cross-sector consultant packs**

| Pack | What it covers |
|---|---|
| [Core](Core) | The 11 universal core skills — the `using-hse-skills` guided router plus risk assessment, JSA, toolbox talk, incident investigation, audits, CAPA, SOPs, RAMS, board reports, incident rates. |
| [Operations](Operations) | 11 operations / management-system skills — ISO 45001 gap analysis, ISO 22301 business continuity, legal & compliance register, regulatory returns, contractor prequalification, emergency-response & induction packs, training-needs / psychosocial / occupational-health assessments. |
| [Leadership](Leadership) | 6 leadership & culture skills — ISO 45001 cl. 5.2 policy, leading/lagging KPI framework, safety-culture assessment, behaviour-based-safety design, leadership/gemba walks, annual ESG OH&S disclosure. |

**The original five sector packs**

| Pack | What it covers |
|---|---|
| [Process Safety](Process-Safety) | 12 process-safety & barrier tools — PHA (HAZID/HAZOP/What-If/LOPA/bow-tie), PSM, PtW, MOC, mechanical integrity, COMAH, PESO. |
| [Chemicals](Chemicals) | 12 chemicals & MAH skills — GHS/SDS, exposure register, transport safety, tank-farm bunding, dispersion & dust-explosion scenarios, India MSIHC/MAH, plus shared PHA tools. |
| [India Compliance](India-Compliance) | 5 legacy-first India skills — state-form finder, Factories Act returns, accident notice, BOCW, OSH Code pack. |

| [Mining](Mining) | 7 mining skills — DGMS statutory pack, ICMM critical controls, PHMP, ventilation/strata/blasting plan, mine-rescue ERP, mine incident investigation, plus the shared bow-tie. |

## Extending the pack

Want to build a new skill or change an existing one? Use **`hse-skill-forge`**, the contributor build tool that scaffolds born-conformant skills. It ships in the separate `hse-systems` bundle and is not one of the normal consultant artifact cards in this manual. Follow [`CONTRIBUTING.md`](../CONTRIBUTING.md) and the authoring docs, then update the relevant pack page under `docs/wiki/` so the manual stays aligned with the skills it describes.
