# India Compliance

The **India compliance pack** (`hse-india`) is a **legacy-first**, **state-detection-mandatory** set of skills that resolve and assemble the statutory safety paperwork an Indian establishment actually files today — the prescribed *state* form, its citing rule, its due date and its filing portal — never a hard-coded national form number. Install it if you are an HSE manager or consultant working under the Factories Act, the BOCW Act, or the (transitioning) OSH Code 2020 in India, where the operative return is administered state-by-state and a wrong state is a wrong statutory form. Every skill resolves a value from the knowledge-base row or marks it a literal `[GAP]` — it refuses to invent a form number — and appends the OSH-Code direction-of-travel note.

```
/plugin install hse-india@hse-leadership-skills
```

**Audience legend:** **M** = manager · **E** = executive · **F** = frontline · **C** = consultant.

### bocw-compliance
- **Produces:** a BOCW (Building & Other Construction Workers Act 1996) statutory-compliance gap-list for a named construction establishment in a given state — registration, the 1% welfare cess, the Form XXV annual return, Safety-Officer/Committee thresholds, and accident notification.
- **For:** M, C · **Grounded in:** BOCW Act 1996 + Rules (`KB-REG-IN-BOCW`); state Welfare Board returns/portal via `KB-REG-IN-STATEFORMS` / `KB-REG-IN-PORTALS`; India, legacy-first, state-detection mandatory · **Packs:** hse-india
- **Use when:** a manager or consultant asks "are we BOCW-compliant in [state]?" — you need to know which establishment/worker registrations apply, the cess obligation, which Welfare Board return is filed and when (Form XXV, ~15 Feb), the Safety-Officer/Committee thresholds, and the accident-notice duty.
- **Don't use for:** assembling a Factories-Act return — use [factories-act-returns](#factories-act-returns); merely *finding* which form applies — use [india-state-form-finder](#india-state-form-finder).
- **Have ready:** the **state** (a blocking gate, asked first — the BOCW return/portal is state Welfare Board specific); peak construction headcount and the direct-vs-contract split; the **cost of construction** (drives the 1% cess); whether you are the principal employer or a contractor (who registers, who pays cess); what's already in place (the gap-list is computed against it); and your filing-cycle position.
- **Trigger:** "are we BOCW-compliant in Tamil Nadu?", "which BOCW welfare-board return do we file and when?", "BOCW cess and Form XXV gap-list for our site".
- **You get:** a branded DOCX + PDF compliance verdict / gap-list — each gap carrying its cited rule, a hierarchy-of-controls-ranked corrective action with a named owner and a due date, every unverified state-specific value flagged `[GAP]`, plus the OSH-Code (BOCW subsumed) transition note and the state Welfare Board portal pointer. Decision-support only; a competent person must review it.
- **Pairs well with:** [india-state-form-finder](#india-state-form-finder) (find the exact form first), [factories-act-returns](#factories-act-returns), and [india-accident-notice](#india-accident-notice) for the construction accident notification.

### factories-act-returns
- **Produces:** the filled India **Factories Act state statutory return / register** (annual or half-yearly) for a detected state — with the prescribed form id, citing rule, due date and filing portal.
- **For:** M, C · **Grounded in:** Factories Act 1948 + state Factory Rules (`KB-REG-IN-STATEFORMS` — TN Form 22 / KA Form 20 / MH Form 27 / DL Form 21; GJ `[GAP]`); India, legacy-first, state-detection mandatory · **Packs:** hse-india
- **Use when:** a manager or consultant says "assemble our [state] Factories Act annual return" and you need the actual filled state return or statutory register, not just a pointer to which form applies.
- **Don't use for:** merely *finding* which form applies — use [india-state-form-finder](#india-state-form-finder); the accident notice specifically — use [india-accident-notice](#india-accident-notice); the OSH-Code mapping — use [india-osh-code-pack](#india-osh-code-pack).
- **Have ready:** the **state** (a blocking gate, asked first — a wrong state is a wrong statutory form; an unseeded state yields `[GAP]` and a refusal to invent a national form number); the return fields field-group by field-group (accident tally by class, leave-with-wages, OSH appointments); average/maximum workers and man-days (drive the thresholds and the return body); the named **occupier / factory manager** who signs; the return year/half; and whether this is a first filing or a correction.
- **Trigger:** "assemble our Maharashtra Factories Act annual return", "fill the TN Form 22 half-yearly return", "Karnataka factory statutory register for our plant".
- **You get:** a branded DOCX + PDF of the assembled return under its prescribed `form` / `rule` / `due`, every field absent from the source carried as a literal `[GAP]` (never fabricated — including the MH annual-return form-id, which the KB does not seed), plus the OSH-Code transition note, a pointer to [india-osh-code-pack](#india-osh-code-pack), and the state factory-department portal pointer. Decision-support only; a competent person must review it before filing.
- **Pairs well with:** [india-state-form-finder](#india-state-form-finder) (resolve the form first), [india-osh-code-pack](#india-osh-code-pack) (the consolidated direction of travel), and [bocw-compliance](#bocw-compliance) for construction establishments.

### india-accident-notice
- **Produces:** the filled India **statutory accident / dangerous-occurrence notification** for a detected state — the legacy accident form + deadline + register entry, after an incident.
- **For:** M, C · **Grounded in:** Factories Act state Rules (`KB-REG-IN-STATEFORMS` — e.g. MH Form 24 within 24h + Form 24A; TN Form 18 + Form 26 register; DL Form 23/27) and the Mines/DGMS layer (`KB-REG-IN-MINES-ACT` / `KB-REG-IN-DGMS` — the 24h notice + Form J register); India, legacy-first, state-detection mandatory · **Packs:** hse-india
- **Use when:** a manager or consultant says "we had an accident in [state] — what do we file and by when?" and you need the assembled notification (which form, the 24h deadline, the register entry, and the filing authority/portal), including for a mine (DGMS notice).
- **Don't use for:** the root-cause investigation / RCA — use [incident-investigation](Core#incident-investigation) (this skill points back to it and does not re-run the analysis).
- **Have ready:** the **state** (a blocking gate, asked first; for a mine, the DGMS region/zone is a *second* mandatory gate); the incident **severity / class** (fatal / serious / dangerous occurrence — it sets the form and the deadline); the **exact incident date-time** that starts the 24-hour clock (preserved, not scrubbed); the statutory notifier / signatory; persons injured or killed (counts only); and the records held.
- **Trigger:** "we had a fatal accident in Maharashtra — what do we notify and by when?", "fill the Form 24 24-hour accident notice", "dangerous-occurrence notification for our factory in [state]".
- **You get:** a branded DOCX + PDF of the filled notification under its prescribed `form` / `rule` / `due`, the load-bearing 24h timing validated with a named owner and deadline (`smart_actions`), every unverified state/form id (e.g. TN/DL accident forms, the mine Form J) carried as a literal `[GAP]`, plus the OSH-Code transition note, the portal pointer, and a hand-off to [incident-investigation](Core#incident-investigation) for the RCA. Decision-support only; a competent person must review it before filing.
- **Pairs well with:** [incident-investigation](Core#incident-investigation) (run the RCA), [india-state-form-finder](#india-state-form-finder), and [factories-act-returns](#factories-act-returns) (carry the accident tally into the annual return).

### india-osh-code-pack
- **Produces:** a forward-looking **OSH Code 2020 transition briefing** — mapping an establishment's legacy statutory regime to its consolidated OSH-Code equivalent (single registration, single consolidated annual return, the raised factory thresholds, the shifted Safety-Officer trigger) with a per-state commencement caveat.
- **For:** M, E, C · **Grounded in:** OSH Code 2020 (`KB-REG-IN-OSH-CODE`) mapped from the legacy state regime (`KB-REG-IN-STATEFORMS`); India, legacy-first, state-detection mandatory; `status: beta` — the law is actively transitioning · **Packs:** hse-india
- **Use when:** someone asks "how will the OSH Code 2020 change what we file in [state], and is it live yet?" — you want the legacy-first answer plus the consolidated direction of travel and an honest per-state "is it notified yet?" caveat.
- **Don't use for:** finding or assembling the legacy form itself — use [india-state-form-finder](#india-state-form-finder) / [factories-act-returns](#factories-act-returns). This is an opt-in transition mode, not the everyday filing skill.
- **Have ready:** the **state** (a blocking gate, asked first — it drives whether the state has notified its OSH Rules; unseeded/Unknown yields direction-of-travel only, never a state-specific consolidated form); the establishment type (each legacy regime maps differently); worker headcount + power (drives the threshold-in/out conclusion — factory 10/20→20/40, Safety-Officer 1000→500/250); what you file today (the *from* side of the mapping); and the transition-mode toggle (legacy-only vs legacy + mapping).
- **Trigger:** "how does the OSH Code change our filings in Karnataka?", "is the OSH Code 2020 live in [state] yet?", "map our legacy Factories-Act returns to the OSH Code equivalent".
- **You get:** a branded DOCX + PDF that leads with the legacy-first answer, then maps each legacy obligation to its consolidated OSH-Code equivalent and flags what changes — with any form in an unnotified state rendered `[GAP]` (never instructing you to file an OSH form your state has not notified) and the savings-clause note that legacy filings remain valid. Decision-support only; a competent person must review it.
- **Pairs well with:** [india-state-form-finder](#india-state-form-finder) and [factories-act-returns](#factories-act-returns) (the legacy *from* side), and [bocw-compliance](#bocw-compliance) (BOCW is among the regimes the Code subsumes).

### india-state-form-finder
- **Produces:** the resolved **legacy state form + citing rule + due date + filing portal** for an India statutory obligation `(law, state, obligation)` — the legacy-first state-form engine.
- **For:** M, C · **Grounded in:** state Factory / BOCW Rules via `KB-REG-IN-STATEFORMS` (TN Form 22 / KA Form 20 / MH Form 27 / DL Form 21 / GJ `[GAP]`; BOCW Form XXV) with portals via `KB-REG-IN-PORTALS`; India, legacy-first, state-detection mandatory · **Packs:** hse-india
- **Use when:** a manager or consultant asks "what form do we file in [state] for [obligation]?" — you need to know *which* form applies, *when* it is due, the *rule* that prescribes it, and *where* it is filed.
- **Don't use for:** assembling the filled return — use [factories-act-returns](#factories-act-returns); running an RCA — use [incident-investigation](Core#incident-investigation); the OSH-Code transition mapping — use [india-osh-code-pack](#india-osh-code-pack).
- **Have ready:** whether you want to *identify* or *assemble* (scope); the **state** (a blocking gate, asked first, infer-from-address-then-confirm — a wrong state is a wrong statutory form; an unseeded state yields `[GAP]` and a refusal to invent a national form); the establishment type (a mine routes to DGMS); the obligation; and the filing year / half.
- **Trigger:** "what form do we file in Tamil Nadu for the Factories Act annual return?", "which BOCW return applies in [state] and when is it due?", "find the citing rule and portal for our Delhi factory return".
- **You get:** the matched `KB-REG-IN-STATEFORMS` row returned as the primary legacy-first answer — `form` / `rule` / `due` / `portal` — with a `[GAP]` surfaced explicitly where the row is verified-absent (e.g. the GJ form, never substituted), the row's OSH-transition note appended (pointing to [india-osh-code-pack](#india-osh-code-pack)), and the `KB-REG-IN-PORTALS` pointer, rendered as a branded DOCX + PDF. Decision-support only; a competent person must review it.
- **Pairs well with:** the natural entry point to the pack — hands off to [factories-act-returns](#factories-act-returns) to assemble, [india-accident-notice](#india-accident-notice) after an incident, and [india-osh-code-pack](#india-osh-code-pack) for the transition view.
