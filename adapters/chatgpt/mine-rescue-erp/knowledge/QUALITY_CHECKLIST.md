# Pre-output Quality Checklist — mine-rescue-erp

- [ ] The ERP is **site-specific** — actual mine, scenarios, and rescue resources.
- [ ] The mobilisation is **realistic** — named team/station/mutual-aid + credible timing, not an aspirational placeholder.
- [ ] Each emergency scenario is risk-ranked (`risk_matrix`).
- [ ] The drills schedule has owner + date on every entry (`smart_actions`).
- [ ] The Mines Act/DGMS emergency-preparedness duty is cited.
- [ ] De-identification pass complete BEFORE drafting; no injured-miner identifier leak; no <5 casualty cell published.
- [ ] The output reads as a draft for a competent person's sign-off.
