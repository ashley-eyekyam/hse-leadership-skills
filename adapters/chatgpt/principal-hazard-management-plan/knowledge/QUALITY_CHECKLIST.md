# Pre-output Quality Checklist — principal-hazard-management-plan

- [ ] The principal hazard is named + specific (from the taxonomy).
- [ ] The hazard is risk-rated on the org matrix (`risk_matrix`).
- [ ] The control suite is hierarchy-ranked; **critical controls flagged + linked to ICMM CCM**; no PPE/admin-only treatment of a principal hazard without escalation.
- [ ] Monitoring has owner + due date (`smart_actions`).
- [ ] Any un-supplied engineering value is `[GAP]` — **no fabricated value**.
- [ ] The output reads as **workshop structuring for the team**, never an autonomous plan (assistive framing).
- [ ] De-identification pass complete BEFORE drafting; accountabilities are role labels.
