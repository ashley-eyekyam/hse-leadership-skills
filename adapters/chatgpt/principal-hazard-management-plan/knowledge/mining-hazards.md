# Mining principal-hazard library + benchmark figures (with source+year)

**Fragment ID:** `KB-DATA-MINING-HAZARDS`
**Discipline:** the mining **principal-hazard library** + any benchmark figures —
every figure carries a non-empty **`source` + `year`** (a benchmark is never a
bare number; an unsourced figure is a `[GAP]`, resolved from the cited authority
at use time, never fabricated). Controls for any of these hazards are
hierarchy-ranked (`KB-SNIP-HOC`): a PPE/admin-only treatment of a principal hazard
is flagged, not accepted.

> Volatile: true · last_reviewed: 2026-05-15 (benchmark vintages drift; refresh on review).

---

## Principal-hazard library (the named mining principal hazards)

| Principal hazard | Material unwanted event (example) | Typical critical-control families |
|---|---|---|
| Ground / strata failure | roof fall, highwall collapse, rib failure | support design, strata monitoring, exclusion, geotech review |
| Inrush / inundation | water/slurry/gas inrush into a working | barrier/standoff, hydrogeological survey, drainage, monitoring |
| Fire / explosion | coal-dust/methane explosion, conveyor/diesel fire | gas monitoring, ventilation, stone-dusting, ignition control |
| Ventilation failure | irrespirable or flammable atmosphere | primary/auxiliary ventilation, gas monitoring, refuge |
| Mobile-plant / vehicle interaction | haul-truck / light-vehicle / pedestrian collision | traffic-management plan, segregation, proximity detection |
| Fall from height | fall from highwall / shaft / raise | edge protection, fall-arrest design, access control |
| Hazardous energy | electrical contact, stored-energy release | isolation / LOTO, design, competence |
| Respirable dust / OH exposure | pneumoconiosis / silicosis from chronic exposure | dust suppression, ventilation, monitoring, health surveillance |

## Benchmark figures (resolve from the cited authority — source+year mandatory)

Mining fatality/injury benchmark figures are **resolved from the cited publishing
body at use time** and quoted as *body + year + measure* — never a bare number.
Where a figure is not resolvable it is a `[GAP]`, never invented.

| Measure | Resolve from (authority) | How to present |
|---|---|---|
| Mining fatality rate (India) | DGMS annual statistics / report | "DGMS annual report, [year]: <rate>" |
| Mining injury / serious-injury rate (India) | DGMS annual statistics / report | "DGMS annual report, [year]: <rate>" |
| Global mining safety performance | ICMM benchmarking / member reporting | "ICMM benchmarking, [year]: <rate>" |

> Source: DGMS annual statistics / reports (India mining fatality & injury rates); ICMM safety-performance benchmarking — per-row authority+year; figure resolved at use time, `[GAP]` where unresolved · Year: 2026.

## How the skills use this fragment
- `icmm-critical-control-management` + `principal-hazard-management-plan` select
  the material unwanted event / principal hazard from the library above.
- `mine-ventilation-strata-blasting-plan` ranks the site hazards against the
  library and the org risk matrix.
- Any benchmark figure cited carries authority+year (or `[GAP]`); a PPE/admin-only
  control for a principal hazard is escalated or justified (`KB-SNIP-HOC`).
