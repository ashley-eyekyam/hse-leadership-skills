# Lone working — HSE INDG73 risk-assessment duty map

**Fragment ID:** `KB-REG-LONE-WORKING`
**What this is:** a copyright-safe **duty → control map** for managing lone-working
risk — grounding `lone-working-assessment` (REN-02). It names the explicit
risk-assessment duty, the reliable-communication / scheduled-check-in / escalation
requirement and the violence + stress/wellbeing considerations.
**What this is NOT:** a reproduction of HSE INDG73 text. Cite the guidance reference
and the duty topics only.

> Source: HSE INDG73 (Protecting lone workers — how to manage the risks of working alone), rev 4 — citation map · Year: 2026 · Reviewed: 2026-06-23 · Volatile: true.

HSE INDG73 sets out how employers manage the risks of working alone. There is **no
general legal ban on lone working**, but an **explicit risk-assessment duty** under
the general HSWA framework. It grounds the lone-working limb of a renewables /
field-work artifact.

## Duty → duty → control it grounds

| Element | Duty (paraphrased) | Control it grounds |
|---|---|---|
| Risk assessment | Assess whether the work can be done safely by one person; some tasks need more than one | lone-working risk assessment |
| Reliable communication | Provide reliable, coverage-checked communication with the lone worker | comms provision |
| Scheduled check-ins | Establish scheduled check-ins and a procedure if a check-in is missed | check-in + escalation procedure |
| Escalation / raise-the-alarm | Define who responds, how, and in what time when contact is lost | escalation/response plan |
| Violence + stress/wellbeing | Consider violence-to-staff risk and the stress/wellbeing of working alone | additional controls |

## The control discipline
Lone-working risk is controlled by a **risk assessment + reliable comms + scheduled
check-ins + a missed-check-in escalation path** — a lone-worker device is a
*supplement*, never the assessment (see `KB-SNIP-CHECKIN-ESCALATION` and
`KB-STD-BS8484`). "No mobile signal" is a control failure, not an accepted risk.
