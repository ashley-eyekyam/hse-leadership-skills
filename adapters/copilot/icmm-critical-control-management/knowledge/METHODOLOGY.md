# Methodology — ICMM Critical Control Management (assistive)

Extends the **B6 safety-audit** verification-activity framing. **Mining-owned** CCM overlay; the bowtie technique is **referenced** from the shared `bowtie-builder` facilitator (hse-process), never re-authored (Decision 3).

## Steps
1. **Intake the material unwanted event** from the mining principal-hazard taxonomy.
2. **Identify the critical controls** — those that, if absent/failed, allow the event; hierarchy-rank (`controls` / `KB-SNIP-HOC`), flag the critical ones.
3. For each critical control set **performance requirement + verification activity + frequency + accountability** (the CCM core).
4. **For the bowtie diagram, reference `bowtie-builder`** by canonical name (`KB-STD-CCPS-BOWTIE`); overlay the mining criticality. Do NOT re-author a bowtie engine.
5. **`[GAP]` not a fabricated value** — where the team has not supplied performance evidence, record `[GAP]`; never invent an engineering value (no-fabrication, D-05a).

Assistive: this structures the multidisciplinary workshop; the mine team owns the judgement.
