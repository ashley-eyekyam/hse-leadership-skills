# Pre-output Quality Checklist — Health & Safety File (CDM 2015 Reg 12(5)–(9))

Before producing any output, validate the draft against this gate. The **residual-only
discipline** and the **Reg 12(5) duty / handover** items are load-bearing — a file that fails
either is not defensible.

## Residual-only discipline (the core-value lever)
- [ ] **Only residual & unusual hazards a future worker could NOT reasonably anticipate are recorded** — a full general-spec dump (the whole specification in place of the residuals) is a specificity/defensibility failure, REJECTED.
- [ ] **Every residual hazard is LOCATED** (where in the structure) — not a generic hazard list.
- [ ] A routine/anticipatable hazard a competent future worker would expect from the structure is NOT included as a residual.
- [ ] If the user states "none beyond the obvious", that justification is recorded explicitly (the gate is met by a stated justification, never by silence).

## Content heads (L153 Appendix 4)
- [ ] **As-built & services information** captured — what was built + use, key as-built drawings, services & isolation-point locations; each cited or `[GAP]`.
- [ ] **Hazardous materials in situ** listed **with locations** (asbestos register / coatings / insulation); an outstanding survey recorded as presumed-present-pending-survey `[GAP]` + a hold-point before disturbance.
- [ ] **Maintenance/cleaning/refurb/demolition safety arrangements** authored for the Q5 future-work scope; **every future-work control HoC-ranked** (`controls`) — no PPE/admin-only without a justified higher-order control.
- [ ] **Information gaps** — every missing as-built source recorded as a `[GAP]`; every open completion item owned + ISO-dated (`smart_actions`).
- [ ] **Revision/version control** present; **an update APPENDS — it never overwrites prior residual-risk content.**

## Regulatory citation (hard-fail dimension)
- [ ] **Preparation is attributed to the principal designer (Reg 12(5))** — attributing it to the wrong duty-holder is a `regulatory_citation_accuracy` hard-fail.
- [ ] **The handover duty is stated** — review/update through the project, **handover to the client at completion**, client retention/availability (Reg 12(6)–(9)); **omitting the handover duty is a `regulatory_citation_accuracy` hard-fail.**
- [ ] The one-line **PCI → CPP → H&S File** (Reg 4 → 12 → 12(5)) cross-reference is present, sourced from `KB-SNIP-CONSTRUCTION-CLAUSE-MAP`, **without assuming a sibling skill ran**.
- [ ] India: the state is resolved before any form; defers to `hse-india`; an un-seeded state → `[GAP]`, **never a minted national form number**.

## De-identification & defensibility
- [ ] De-identification pass complete **BEFORE** drafting; **no commissioning/survey-record name or any health detail** leaked; **no worker health detail in the file**; `<5` cells suppressed; no embedded re-id key.
- [ ] Restricted-distribution flag set where the file carries security-sensitive structural/services detail.
- [ ] The user-supplied principal designer / client duty-holders + open-item owners stay named (legitimate record); commissioning/survey names scrub to role labels.
- [ ] No conclusion rests on an unstated assumption.
- [ ] SME review (CDM Principal Designer persona) ran; it never emits "approved by a competent person".
