# IEC 61511 — Functional Safety / LOPA (method map)

**Fragment ID:** `KB-STD-IEC-61511`
**What this is:** a copyright-safe **method map** for Layer of Protection Analysis
(LOPA) and the functional-safety / Safety Integrity Level (SIL) allocation
structure — initiating event → independent protection layers (IPLs) with
independence tests → residual-risk gap → SIL.
**What this is NOT:** a reproduction of the standard's normative text. IEC 61511
is copyrighted; the user holds their own licensed copy. Cite the method structure
and topics only — never paste the standard's wording.

> Source: IEC 61511 functional safety — method structure (user holds the standard) · Year: 2026 · Reviewed: 2026-05-15 · Volatile: false (multi-year revision cycle).

LOPA is an **assistive** method: PFD values and IPL credits are
**engineer-supplied and engineer-verified**, never computed or invented by the
skill. It grounds `lopa-worksheet`.

## Method structure → worksheet columns

| Step | Topic | Worksheet column |
|---|---|---|
| 1 | Scenario / consequence | Scenario · Consequence |
| 2 | Initiating event + frequency | Initiating event (engineer-supplied frequency) |
| 3 | Independent protection layers | IPL list, each with an **independence test** result (genuinely independent, not double-counted) |
| 4 | IPL PFD | PFD per IPL (**engineer-supplied** — `[GAP]` if not provided) |
| 5 | Mitigated frequency / residual gap | Residual-risk gap (`risk_matrix` band: tolerable vs not) |
| 6 | SIL allocation | SIL (engineer-supplied target; the skill records, does not allocate) |

## Assistive discipline (load-bearing)
- The skill **does NOT compute PFDs or allocate SIL** — these are engineer inputs
  it structures and records (Decision 6 / spec §3.5).
- A missing PFD/IPL credit is recorded as `[GAP]`, never a fabricated number.
- IPL independence is the team's judgement; the skill prompts the test but does
  not declare an IPL independent on its own.
