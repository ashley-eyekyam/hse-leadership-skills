# Level crossings & track work — closure/separation-first control spine

**Fragment ID:** `KB-SNIP-LX-HIERARCHY`
**This is prompt text, applied by the model — not a generator.** It is the
control-hierarchy spine for `level-crossing-track-worker-safety` (RAIL-03): control
crossing risk by **closure → grade separation → engineering first**, and control
track-worker risk by **separation (green-zone/possession) first** — signage-only or
lookout-only is **never** the primary control where a higher order is reasonably
practicable. A rail artifact that "controls" a crossing with new signage, or a
track task with a lookout alone, is the indefensible paperwork this spine rejects.

## Instruction — the two control orders

**Level-crossing remedial order:**
1. **Closure** (eliminate the crossing).
2. **Grade separation** (bridge/underpass).
3. **Engineering** (barriers / obstacle detection / MSL / full-barrier).
4. **Sighting / signage / administrative** (last resort).

**Track-worker protection order:**
1. **Separation** (green-zone / line blockage / possession).
2. **Safe systems of work (SSOW)**.
3. **Warning** (TOWS / other warning systems).
4. **Lookout-only** (last resort).

## The gate (reject these)
- A crossing treatment that leads with signage/admin while closure / grade
  separation / engineering is reasonably practicable → **reject**.
- A track task whose lead control is lookout-only while separation (green-zone /
  possession) is reasonably practicable → **reject**.
- A licensed ALCRM band hard-coded with numeric thresholds → **reject**; record the
  user's band, leave thresholds `[GAP]`.
- COSS/lookout persons named instead of role-labelled → **reject**; de-identify.

## How the skill uses this fragment
RAIL-03 ranks every crossing and track-work control through these two orders and
rejects a signage-only / lookout-only-led treatment.
