# ISO 12100:2010 + ISO 14120:2015 — machine safety design + guard taxonomy (method→artifact map)

**Fragment ID:** `KB-STD-ISO12100-14120`
**What this is:** a copyright-safe **method map** for ISO 12100:2010 (Safety of
machinery — general principles for design, risk assessment and risk reduction) and ISO
14120:2015 (Safety of machinery — Guards). It grounds a machine-guarding skill with the
**three-step method** (inherent safe design → safeguarding → information for use) and
the guard-type taxonomy (fixed / movable / interlocking guards).
**What this is NOT:** a reproduction of either standard. Cite the standard number and
the method/taxonomy topics only — never paste the standard wording.

> Source: ISO 12100:2010 + ISO 14120:2015 — method/clause structure (user holds the licensed standards) · Year: 2026 · Reviewed: 2026-06-22 · Volatile: false (multi-year cycle; refresh on review).

ISO 12100 is the type-A machine-safety standard; ISO 14120 is the guard-design
standard. Together they ground a machine-guarding-assessment skill with an
**engineering-control-led** treatment, applied through the hierarchy of controls.

## Method / topic → standard → artifact it grounds

| Method / topic | Standard | Artifact it grounds |
|---|---|---|
| Step 1 — inherent safe design | ISO 12100:2010 §6.2 | eliminate/reduce the hazard by design (remove the nip point, limit force/energy) before any guard |
| Step 2 — safeguarding & complementary protective measures | ISO 12100:2010 §6.3 | guards and protective devices for residual mechanical hazards (point of operation, transmission, rotating parts) |
| Step 3 — information for use | ISO 12100:2010 §6.4 | residual-risk warnings, safe-use instructions, training — the **last** step, never a substitute for steps 1–2 |
| Risk assessment & reduction loop | ISO 12100:2010 §5 | iterate the assessment until residual risk is acceptable |
| Guard-type taxonomy | ISO 14120:2015 | select **fixed / movable / interlocking** guards by access frequency and the hazard (see `KB-SNIP-GUARD-SELECTION`) |

## How the skill uses this fragment
- `machine-guarding-assessment` resolves the hazard per machine zone, then applies the
  three-step method — design-out first, guard the residual, inform last.
- The guard **type** is selected via `KB-SNIP-GUARD-SELECTION` against this taxonomy;
  the US-jurisdiction grounding is `KB-REG-OSHA1910-O`.
- "Operators to be careful / wear gloves" as a headline control is rejected — guarding
  is engineering-control-led (`KB-SNIP-HOC`). A missing input is a `[GAP]`.
