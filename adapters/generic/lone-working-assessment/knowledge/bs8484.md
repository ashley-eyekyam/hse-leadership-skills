# BS 8484:2022 — lone-worker device services map

**Fragment ID:** `KB-STD-BS8484`
**What this is:** a copyright-safe **device-service-conformance map** for lone-worker
device services — grounding `lone-working-assessment` (REN-02). It names the
device + monitored-response-chain conformance and the discipline that a device
*supplements* the procedure, never replaces it.
**What this is NOT:** a reproduction of BS 8484:2022. Cite the standard number and
the topics only.

> Source: BS 8484:2022 (Provision of lone worker device services) — citation map · Year: 2026 · Reviewed: 2026-06-23 · Volatile: false.

BS 8484:2022 sets the requirements for lone-worker device services — the device, the
alarm-receiving/monitoring centre and the response chain. It grounds the
device-service limb of a lone-working artifact.

## Element → requirement → artifact it grounds

| Element | Requirement (paraphrased) | Artifact it grounds |
|---|---|---|
| Lone-worker device | A conformant device able to raise an alarm (manual + automatic where appropriate) | device specification |
| Monitored response chain | An alarm-receiving / monitoring service with a defined response | response-chain specification |
| Conformance | The device + monitoring service conform to BS 8484 | conformance record |

## The discipline
A BS 8484 device **supplements, never replaces, the check-in / escalation
procedure** (`KB-SNIP-CHECKIN-ESCALATION`). A lone-working RA that treats a device
as the whole control — with no scheduled check-in or escalation procedure — is the
indefensible output this map rejects.

## How the skill uses this fragment
REN-02 grounds the lone-worker device service on BS 8484:2022 conformance, layered
**on top of** the check-in/escalation procedure (not instead of it). The specific
device/service the site uses is `[GAP]` until supplied.
