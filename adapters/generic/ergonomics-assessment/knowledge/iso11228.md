# ISO 11228-1/-2/-3 ‚Äî manual handling method scope + handling decision flow (method‚Üíartifact map)

**Fragment ID:** `KB-STD-ISO11228`
**What this is:** a copyright-safe **method map** for the ISO 11228 manual-handling
series ‚Äî Part 1 (lifting and carrying), Part 2 (pushing and pulling), Part 3
(handling of low loads at high frequency / repetitive handling) ‚Äî plus the decision
flow that selects which part applies to a given task. It grounds an ergonomics
skill with the method scope.
**What this is NOT:** a reproduction of the standard, and it carries **no** RULA / REBA
/ NIOSH action bands ‚Äî those are the locked tables in the `ergonomics.py` engine (D-02),
the single scores source. Cite the standard number and the method scope only.

## Method / topic ‚Üí standard part ‚Üí artifact it grounds

| Handling task | ISO 11228 part | Method it grounds |
|---|---|---|
| Lifting & carrying | ISO 11228-1 | the lifting/carrying assessment scope; the engine computes NIOSH RWL / Lifting Index |
| Pushing & pulling | ISO 11228-2 | the push/pull force-and-distance assessment scope |
| Repetitive handling (low load, high frequency) | ISO 11228-3 | the repetitive-handling / upper-limb load assessment scope |

## The handling decision flow
1. Characterise the task (load mass, frequency, posture, push/pull, repetition).
2. Select the **applicable ISO 11228 part(s)** by the handling type above.
3. Run the relevant assessment through the **`ergonomics.py` engine** (RULA / REBA /
   NIOSH RWL) ‚Äî same inputs ‚Üí same score; the bands are the engine's, cited there.
4. Rank controls via `KB-SNIP-ERGO-CONTROLS` (work-design-first; training is **not** a
   control for biomechanical overload).

## How the skill uses this fragment
- `ergonomics-assessment` resolves the handling type, selects the ISO 11228 part for
  the **method scope**, and reads the action bands from the engine ‚Äî not from here.
- A missing input (no load mass, no frequency) is a `[GAP]`; an unconfirmed posture is
  never scored.
