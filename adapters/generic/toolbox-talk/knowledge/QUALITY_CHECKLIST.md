# Pre-output Quality Checklist — toolbox talk

Validate the assembled talk against this 5-point gate **before** producing the output
(the mandatory Critic/QA self-check). The five items map 1:1 to the eval expectations
(§3.11) — each is a thing the eval rubric scores.

- [ ] **(a) Specific, not generic.** The talk names *this* task, *this* site/area, and
      *this* crew, and the concrete observable hazards of the named job — not a generic
      hazard category ("work safely at height"). If the intake gave only a vague task,
      it was refused or recorded `[GAP]`, never papered over. → eval case 1
      (`specificity`).
- [ ] **(b) Controls hierarchy-of-controls-ranked.** Higher-order controls are stated
      first; any PPE/admin-only treatment carries an explicit "higher-order controls not
      reasonably practicable because…" justification — never a bare "wear your PPE".
      Each control names the hazard it addresses. → eval case 1 (`hierarchy_of_controls`).
- [ ] **(c) Incident de-identified or labelled-typical — never fabricated-as-local.**
      Any referenced real incident uses role labels only (no name, no <5 cell, no
      re-identification key). When no incident was supplied, a clearly-labelled
      typical/illustrative example is used — there is **no** fabricated "happened on this
      site" event. → eval case 2 (`de_identification` auto-fail).
- [ ] **(d) Within the duration target and frontline-readable.** The talk is **short**
      (within the Q4 duration target — not a multi-page document), plain-language, and at
      the crew's reading level (Q6); short paragraphs, structured hook → hazards →
      controls → incident → discussion → takeaways → sign-off. → eval case 3 (length /
      readability).
- [ ] **(e) Sign-off sheet present.** The output includes a fillable attendance /
      sign-off table (Name/Role · Signature · Date) — the auditable record. → eval case 3
      (sign-off present).
